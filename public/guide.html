ORB API Endpoints

Public Endpoints

POST /auth

/auth works as an endpont for user authorization.

Requires a username and password as x-www-form-urlencoded parameters. Returns an authorizaton token as a JSON response.

Parameter Value
name username (String)
password password (String)

An example response: { "success": true, "message": "You are tokenized!", "token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyOi..." }

Authorized Endpoints

All authorized endpoints require an authorization token (JWT) for access. The token can be claimed from /auth endpoint by logging in. Send the acquired token as a Authorzation: Bearer <token> HTTP Header for the following endpoints.

GET /api

API root.

GET /api/users/

Returns all the users on the database. Currently no marshalling is done.

POST /api/users

Creates a single user.

Parameters need to be given as x-www-form-urlencoded type.

Parameter Value
name username (String)
password password (String)
age age (Number)
gender male | female (String)
note public note (String)
latitude latitude coordinates (Number)
longitude longitude coordinates (Number)

An example response: { "success": true }
